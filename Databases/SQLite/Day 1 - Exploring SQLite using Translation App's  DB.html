How to run SQLite on Windows?

To run SQLite on Windows, follow these steps:

1. Download SQLite

  1. Go to the official SQLite website:
    🔗 https://www.sqlite.org/download.html
  2. Scroll down to the "Precompiled Binaries for Windows" section.
  3. Download "SQLite Tools for Windows" (ZIP file).
  4. Extract the ZIP file to a folder (e.g., C:\sqlite).

2. Add SQLite to System PATH (Optional, for easier access)

  1. Copy the extracted folder path (e.g., C:\sqlite).
  2. Search for "Environment Variables" in Windows.
  3. Under System Variables, find and edit the Path variable.
  4. Click New and paste C:\sqlite.
  5. Click OK to save changes.

Now, you can run sqlite3 from any command prompt.

3. Run SQLite

Option 1: Using Command Prompt

  1. Open Command Prompt (cmd).
  2. Navigate to the SQLite folder (if you didn't add it to PATH):
    sh
    cd C:\sqlite
  3. Start SQLite by running:
    sh
    sqlite3
  4. To open an existing database or create a new one:
    sh
    sqlite3 my_database.db
  5. To check tables in the database:
    sql
    .tables
  6. To exit SQLite:
    sh
    .exit

Option 2: Using SQLite GUI Tools (Optional)

If you prefer a graphical interface, you can download a GUI tool:

  • DB Browser for SQLite: https://sqlitebrowser.org/

This tool allows you to easily create and manage SQLite databases without using the command line.

Option 3: Using SQLite with Python

If you want to interact with SQLite in Python, use the sqlite3 module:

python
import sqlite3

# Connect to a database (or create one)
conn = sqlite3.connect("my_database.db")

# Create a cursor
cursor = conn.cursor()

# Create a table
cursor.execute("CREATE TABLE IF NOT EXISTS users (id INTEGER PRIMARY KEY, name TEXT);")

# Insert data
cursor.execute("INSERT INTO users (name) VALUES ('Alice');")
conn.commit()

# Query data
cursor.execute("SELECT * FROM users;")
print(cursor.fetchall())

# Close connection
conn.close()

Show all tables.

To show all tables in an SQLite database, use the following SQL query:

sql
SELECT name FROM sqlite_master WHERE type='table';

Explanation:

  • sqlite_master: This is a system table in SQLite that stores information about all database objects (tables, indexes, views, triggers).
  • WHERE type='table': Filters the results to show only tables (excluding indexes, views, etc.).

Alternative (Using .tables in SQLite CLI)

If you're using the SQLite command-line interface, you can simply run:

sh
.tables

This will list all the tables in the current database.

Get table schema using SQL.

To get the schema of a table in SQLite, you can use the following SQL command:

sql
PRAGMA table_info(your_table_name);

Explanation:

  • PRAGMA table_info(your_table_name): Returns details about the table structure, including:
    • Column ID (cid)
    • Column Name (name)
    • Data Type (type)
    • Nullable Constraint (notnull)
    • Default Value (dflt_value)
    • Primary Key Status (pk)

Example Usage:

sql
PRAGMA table_info(HindiToEnglishApp);

Output Example:

pgsql
cid | name       | type      | notnull | dflt_value | pk
----+-----------+----------+--------+------------+----
0   | id        | INTEGER  | 1      | NULL       | 1
1   | word      | TEXT     | 1      | NULL       | 0
2   | meaning   | TEXT     | 0      | NULL       | 0

Alternative: Using sqlite_master

To get the CREATE TABLE statement that defines a table's schema, use:

sql
SELECT sql FROM sqlite_master WHERE type='table' AND name='your_table_name';

Example:

sql
SELECT sql FROM sqlite_master WHERE type='table' AND name='HindiToEnglishApp';

Output Example:

sql
CREATE TABLE HindiToEnglishApp (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    word TEXT NOT NULL,
    meaning TEXT
);

This query retrieves the exact SQL statement that was used to create the table.
