Here’s the quick breakdown:

  • armv8l → ARMv8 architecture, little‑endian, but the kernel/OS is presenting a 32‑bit userspace (the l stands for “little‑endian” and armv8 with a lower‑case v typically indicates the 32‑bit execution state of ARMv8, i.e., AArch32).

  • aarch64 → the same chip would show this if running a full 64‑bit OS/kernel.

  • armv7l → actual 32‑bit only ARMv7 chip.

So your phone has a 64‑bit capable CPU (ARMv8 or later) but the current Android/ROM is booting a 32‑bit kernel or your Termux is in a 32‑bit mode. Most Android devices from the last 7+ years are 64‑bit capable, even if uname -m shows armv8l.

How to use this when comparing to a new phone
